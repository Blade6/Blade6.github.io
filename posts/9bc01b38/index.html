内部类

发表于 2017-10-11 | 分类于 Java | 阅读次数

内部类
以下四种内部类都可以在其内部在定义内部类，至于这些内内部类将视为内部类的成员变量或方法代码块，受到成员变量的限制或方法的限制。比如，匿名内部类里不能有static变量，所以其内部不能有嵌套内部类，而嵌套内部类允许内部有嵌套内部类。

成员内部类

作为外部类的一个成员存在。
成员内部类可以访问它的外部类的所有成员变量和方法，不管是静态的还是非静态的。访问静态变量则“类名.变量”，访问非静态变量则使用“类名.this.变量”，方法同理。
在外部类里面创建成员内部类的实例可以直接new memberClass()或者this.new memberClass()。如果在外部类之外创建内部类的实例则需要先创建外部类的实例，然后再创建成员内部类的实例，如(new outerClass()).new memberClass()。
成员内部类中不能有静态方法，也不能有静态变量，同时也不允许静态代码块。但是可以有静态常量，也就是static和final同时出现修饰变量。
可以是抽象类。

静态内部类

比成员内部类多了一个static修饰，又称嵌套内部类。
静态内部类只能访问它的外部类的静态变量和静态方法。外部类的变量和方法通过类名.变量、类名.方法只能访问静态内部类的静态变量和静态方法，通过new构建实例可以访问静态内部类的非静态变量和非静态方法。
静态内部类中可以有静态变量、方法，也可以有非静态变量、方法，还可以有静态代码块。
这里说一句，对于内部类，Java虚拟机会生成对应的Class文件，因此实际上静态内部类作为一个类，有和其所在外部类隔离的地方，体现在比如静态内部类有静态代码块，但是该静态代码块可以读取出现在其后面定义的外部的静态变量的值，外部类的静态代码块也可以读取定义在其后面的静态内部类中的静态变量的值。
可以是抽象类。

局部内部类

又称方法内部类，定义在方法中，作用域就在方法内部。
像局部变量一样，不能被public, protected, private和static修饰。但可以被final或abstract修饰。
不能有静态变量或方法，可以有静态常量。
可以是抽象类，可以有抽象方法。
局部内部类内部是否能访问外部类的静态变量或方法取决于该局部内部类在静态方法中还是在非静态方法中。
局部内部类只能访问所在方法定义的常量，而不能访问所在方法中定义的变量。
这里多说一句，Java局部变量只允许被final修饰。

匿名内部类

匿名内部类就是没有名字的内部类。
匿名内部类为局部内部类，所以局部内部类的所有限制同样对匿名内部类生效。换言之，局部内部类不能做的事情，匿名内部类也不能做，但是局部内部类能做的事情，匿名内部类未必能做。
在使用的时候都是new parentClass(){}或new parentInterface(){}的形式，所以它没有自己的名字，因此其内部不能有构造函数。

匿名内部类中的方法不能是抽象的，匿名内部类必须实现它的父类或接口中的所有抽象方法。在创建匿名内部类的时候，已经通过new创建实例了，自然不能是抽象的。
匿名内部类会继承一个父类（有且只有一个）或实现一个接口（有且只有一个）。
匿名内部类不能定义任何静态成员变量、方法、代码块，但可以定义静态成员常量。
匿名内部类内部是否能访问外部类的静态变量或方法取决于该匿名内部类在静态方法中还是在非静态方法中。

匿名内部类如果所在的方法有形参，而且在匿名内部类里使用了该形参，则该形参必然是final类型的，否则编译不通过（因为Android中经常使用匿名内部类，所以这一条，Android开发者非常了解）：

而这又是为什么呢？
首先我们知道在内部类编译成功后，它会产生一个class文件，该class文件与外部类并不是同一class文件，仅仅只保留对外部类的引用。当外部类传入的参数需要被内部类调用时，从Java程序的角度来看是直接被调用，实际上是内部类利用自身的构造器对传入的参数进行备份，自己内部方法调用的实际上是自己的属性而不是外部方法传递进来的参数。
在内部类中的属性和外部方法的参数两者从外表上看是同一个东西，但实际上却不是，所以他们两者是可以任意变化的，也就是说在内部类中我对属性的改变并不会影响到外部的形参，然而这从程序员的角度来看这是不可行的，毕竟站在程序的角度来看这两个根本就是同一个，如果内部类里改变了，而外部方法的形参却没有改变这是难以理解和不可接受的，所以为了保持参数的一致性，就规定使用final来避免形参的不改变。
简而言之，拷贝引用，为了避免引用值发生改变，例如被外部类的方法修改等，而导致内部类得到的值不一致，于是用final来让该引用不可改变。故如果定义了一个匿名内部类，并且希望它使用一个其外部定义的参数，那么编译器会要求该参数引用是final的。
